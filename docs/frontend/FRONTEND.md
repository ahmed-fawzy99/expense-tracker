# Frontend — React Module Reference

Single-page React 19 app served by Vite. Talks to Convex directly via React hooks (`useQuery`, `useMutation`). Auth lives entirely server-side; the client only **gates the UI** (hides nav, renders 403 panels in place — never tries to navigate to a forbidden route and bounce).

> Update this file whenever you add a route, page, or shared component.

## Auth + permission gating

1. `__root` decides between three states using Convex's `<Authenticated>` / `<Unauthenticated>` / `<AuthLoading>` boundary components.
2. Pages that require a permission wrap content in `<RequirePermission permission="...">` — the panel renders **in place**. URL never changes on a 403.
3. Pages that require domain state (e.g. `/new` requires `managerId !== null`) check that themselves and render `<ForbiddenPanel>` with a useful message. Same in-place rule.

## Forms

Every form uses `react-hook-form` with a zod v4 schema, wired through
`@hookform/resolvers/standard-schema` (zod v4 implements standard-schema, so
the dedicated zod resolver is no longer used). Types are inferred via
`z.infer<typeof schema>`. Errors render via the shadcn `<Form>` primitives —
**never** as toasts for missing-required-field cases. Toasts are reserved for
operation results (success / unexpected failure).

## Tables

`<DataTable>` is the single generic table used by every list page. It is
**server-paginated, server-sorted, and server-filtered** — the route owns
the Convex query, the table owns the UI shell.

- **Pagination**: cursor-based on the server (`paginationOptsValidator`),
  page-numbered on the client. The cursor history stack lives in the URL
  (`cursors[]`) so a hard refresh on page 3 lands on page 3, not page 1.
  Page size is selectable (10/25/50/100); the default is 10.
- **Sorting**: clicking a sortable header emits `sortBy` + `sortDir` into
  the URL and resets pagination. Backed by `.order(dir)` on the chosen
  Convex index.
- **Filters**: rendered by the route in the `filters` slot above the
  table. Filter changes reset pagination. Status / category / date /
  state are all serialized into the URL search.
- **Row links**: `rowLink(row)` returns TanStack Router `<Link>` props
  so right-click → "Open in new tab" works as expected.

`tableSearchSchema` (in `src/lib/table-search.ts`) provides the base zod
shape every table route extends with its own filter fields, then passes
to `validateSearch` on the route.

## Theme

Paper theme — see [`docs/design/DESIGN.md`](../design/DESIGN.md) for tokens and

(no pure white/black), Roboto Flex body, Montserrat headings, PT Mono code.

## Out of scope for v1 UI

- Admin pages (users CRUD, role assignment, team settings).
- Chain handoff button — backend ships, UI deferred.
- Multi-team switching.
- In-app notification bell — backend ships (see `docs/backend/BACKEND.md` §`notifications.ts`), UI deferred to a future release.
- Email notifications, FX, reimbursement.

## Tests + Deploy

Frontend tests live in `src/__tests__/`, mirroring the backend's `convex/__tests__/`. The suite covers `lib/` (pure logic), `hooks/`, every shared component, and a smoke test per route. See `docs/frontend/patterns.md` §14 for the `mockConvex` / `mockRouter` helper recipe.

Run with `pnpm test:frontend` (this project only) or `pnpm test` (both projects).

`pnpm deploy` is gated by tests — `predeploy` runs `pnpm lint && pnpm test`, so a broken component test blocks the production deploy before Convex or the Vite build runs. Never bypass.
